3SAT → SUBSET-SUM · the digit-position trick

Build numbers whose digits encode variables and clauses; a subset sums to target iff formula is satisfiable.

CLO-4R2·30 ptsL12
1 · Theory
Construction

For n variables, m clauses, build 2n + 2m numbers in base 10: n digits for variables (one bit each) + m digits for clauses (clause-satisfaction count).

Target has 1s in variable positions (exactly one chosen per variable) and 3s in clause positions (clause covered, padded by slack).

3 · Q&A defence notes
  • Q1. Why base 10 (not binary)?
    Base ≥ 4 avoids carries when columns sum to ≤ 3. Base 10 is convenient for humans.
  • Q2. Why slack numbers?
    A clause may have 1, 2, or 3 true literals. Slacks pad each clause sum to exactly 3 regardless.
  • Q3. Size of the reduction?
    2n + 2m numbers, each n + m digits ⇒ O((n+m)²) total. Polynomial in formula size.
