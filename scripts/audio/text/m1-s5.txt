Moving to context-free grammars, our fourth question is A-C-F-G: given a CFG G and a string w, does G generate w? This is decidable, but it requires a bit more work than the DFA case. The standard algorithm is C-Y-K, named after its inventors Cocke, Younger, and Kasami. It works on grammars in Chomsky normal form, where every rule has either two non-terminals or one terminal on the right-hand side. C-Y-K fills in a triangular parsing table bottom-up: each cell records which non-terminals can derive a particular substring of w. The bottom row handles single characters; higher rows combine results from below using the rules of the grammar. The string is in the language if and only if the start symbol appears in the topmost cell. C-Y-K runs in time cubic in the length of the string. The interactive widget animates the table filling in diagonal by diagonal — a beautiful visual demonstration of dynamic programming applied to language recognition. Note that for general grammars the problem remains polynomial, but the constants and the algorithm are more involved.
