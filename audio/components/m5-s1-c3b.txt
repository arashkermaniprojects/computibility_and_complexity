This figure shows the two equivalent definitions of NP, side by side, with the bridge that connects them.

View one, on the left, is the verifier view. A language L belongs to NP if there exists a deterministic Turing machine V, called the verifier, that runs in polynomial time, and a polynomial-size certificate c, such that x belongs to L if and only if there is SOME c for which V on the pair x, c accepts. Think of c as a "hint" or "witness" — for a SAT problem, c is a candidate truth assignment. For a Hamiltonian path problem, c is a candidate ordering of vertices. The verifier's job is to check the hint efficiently. V is deterministic, so given a particular x and a particular c, V produces a single answer in polynomial time.

View two, on the right, is the nondeterministic view. An NTM N starts on input x and, at every computational step, may branch into multiple possible successor states. This produces a tree of computation paths. The NTM accepts the input if SOME path in the tree ends in the accept state q-accept. N runs in polynomial time means every path has polynomial depth. The figure draws the tree with eight leaves at level three, but only one of them — colored green — reaches q-accept. The other seven, colored red, reject.

The middle panel explains why these two views are equivalent. To turn a verifier V into an NTM N: on input x, N nondeterministically guesses c one symbol at a time using its branching ability, then deterministically simulates V on the pair x, c. Each branching choice corresponds to one symbol of c. If any branch of c leads V to accept, then some path of N accepts. To turn an NTM N into a verifier V: V's input is the pair x, c, where c is interpreted as the list of nondeterministic choices N would have made. V simulates N along that single chosen path and accepts if and only if N would have accepted along it. The accepting path's sequence of choices IS the certificate. They are the same object, in two different notations.

The bottom panel states Sipser's theorem 7.20: NP equals the union over k of NTIME of n to the k, equals the class of languages with polynomial-time verifiers. The two viewpoints are interchangeable — pick whichever makes your proof cleaner. The verifier view is usually easier for showing concrete problems are in NP. The nondeterministic view is cleaner for closure properties and for the proof that SAT is NP-complete.
