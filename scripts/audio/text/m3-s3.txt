Now the negative result for L-B-As: E-L-B-A is undecidable. The question E-L-B-A asks whether the language of an L-B-A is empty. This is undecidable, and the proof uses the computation history trick. Given ⟨M, w⟩, an instance of A-T-M, we construct an L-B-A B that on its input checks whether the input encodes an accepting computation history of M on w. B accepts a string if and only if that string is exactly such a history. So the language of B is empty if and only if M does not accept w. A decider for E-L-B-A would therefore decide A-T-M, which is impossible. The widget animates the construction of B's checking procedure, walking through how local consistency between consecutive configurations is verified using bounded tape. The key technical fact is that the check fits within the linear space bound of an L-B-A — that's why we used an L-B-A rather than a finite automaton here.
