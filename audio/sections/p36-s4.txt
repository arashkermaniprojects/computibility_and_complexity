Let's dive into why TQBF is PSPACE-complete rather than just NP-complete. Think about SAT, which deals with problems having only existential quantification—basically, one layer of guessing. TQBF, on the other hand, opens the door to unbounded alternation of quantifiers—like ∀∃∀∃ and so forth. Each switch between existential and universal quantifiers adds a layer of complexity that NP struggles with, while PSPACE can handle it because it operates with space-bounded computation.

Now, how does recursion in TQBF manage to use polynomial space? It’s all about depth-first evaluation. This method cleverly reuses memory at each recursion level. You only need to keep track of the current path’s partial assignment—just one bit for each variable—and a stack frame for each level. So, you end up using O(n) space, which is pretty efficient.

When we talk about the total size of the game tree, we find there are 2 to the power of n leaves and 2 to the power of n plus one minus one internal nodes. In the worst-case scenario, evaluating this tree takes 2 to the power of n time. TQBF also captures the essence of any finite two-player game with alternating moves, like generalized chess or Go, all of which are known to be PSPACE-complete or even harder.
