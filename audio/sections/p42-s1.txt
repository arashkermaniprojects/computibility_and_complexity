Let’s talk about the PATH problem, which is a classic example in the realm of nondeterministic log-space, or NL for short. PATH focuses on directed graphs, specifically checking if there’s a path from a starting vertex s to a target vertex t. It’s considered the canonical NL problem and is even NL-complete, meaning it’s one of the hardest problems in NL.

Now, imagine a nondeterministic algorithm that starts at s and simply makes guesses about which neighbor to explore next. It keeps track of the number of steps it’s taken with a counter. Each time it checks if it’s reached t, if it has, it accepts. If not, it makes a nondeterministic choice from the neighbors of the current vertex and increments the counter until it’s tried all possibilities.

What’s neat here is that it uses very little memory—just enough to store the current vertex and the counter, both of which require only logarithmic space relative to the number of vertices. This is where nondeterminism shines: it allows the algorithm to explore multiple paths simultaneously without needing to remember all the vertices it has visited, which is something a deterministic approach would struggle with given the same space constraints. So, while deterministic algorithms are limited, the power of nondeterminism lets PATH be efficiently solvable in log-space.
