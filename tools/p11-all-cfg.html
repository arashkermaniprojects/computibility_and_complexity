ALLCFG is undecidable

Build a CFG GM,w that generates everything EXCEPT accepting computation histories of M.

CLO-2R1·30 ptsL7
1 · The crucial inversion · check non-membership, not membership

ALLCFG = {⟨G⟩ : L(G) = Σ*}. Sipser proves it undecidable by reducing from ATM. The construction has a key inversion that students often find confusing: we don't build a CFG that recognises accepting computation histories — that would require a CFG to verify a TM, which is beyond CFG power. Instead we build a CFG that recognises everything except accepting computation histories.

Why this matters

CFGs are good at existential structure: "the string has some part that is bad." They can nondeterministically pick a position and check a local property there. They are bad at universal structure: "every part of the string is good." That requires correlating multiple positions without bounded memory — the canonical pumping-lemma failure.

A valid accepting computation history is a universal structure (every adjacent pair is correct). A broken history is existential (some adjacent pair is wrong). So the CFG checks the broken side.

The construction f(⟨M, w⟩) = ⟨GM,w⟩

GM,w generates string x iff x exhibits at least one of these four failures:

  1. x is not formatted as C1 # C2 # … # Cℓ.
  2. C1 ≠ q0w (wrong start config).
  3. Some adjacent pair (Ci, Ci+1) violates δ.
  4. Cℓ does not contain qaccept.

Then L(GM,w) = Σ* if and only if every string fails at least one check, i.e. no valid accepting history exists for M on w, i.e. M does not accept w. A decider for ALLCFG would decide ¬ATM. Impossible.

2 · The reverse-write trick · why CFGs can match (Ci, Ci+1)

The grammar nondeterministically guesses where a violation occurs, then checks it locally. Crucially: the grammar accepts when it finds a violation. A correctly-formed accepting history goes through the grammar without any violation being detectable — so it is not generated.

4 · Pitfalls students stumble on
  • L(GM,w) = Σ* is the answer. The CFG does generate everything iff M doesn't accept w. Generating everything means "every string is broken." Don't flip the direction in your head.
  • The reverse-write trick is not optional. Without writing alternate configurations backwards, the CFG would have to match c1 with cn+1 at the same position — which requires synchronisation a CFG can't do. The trick converts position-matching into adjacency-matching.
  • The grammar accepts the violation, not the history. Newcomers say "the grammar checks the history is correct." It does the opposite: it generates the wrong histories. The valid histories are the ones the grammar can't produce, and those are exactly the witnesses to ATM.
  • Don't try to make G deterministic. CFGs are inherently nondeterministic; for each candidate violation type, the grammar nondeterministically commits to "this is where the error is." Multiple derivations may exist for the same string — that's fine, it only needs one.
5 · Q&A defence notes
  • Q1. Why is "find ONE violation" easier than "check ALL pairs are correct"?
    Existential vs universal quantification. A CFG can nondeterministically pick a position and locally verify a single property there — that's the pushdown automaton's natural mode of operation. "No problems anywhere" requires correlating all positions simultaneously, which exceeds the CFG/PDA's bounded-stack capability. This existential/universal asymmetry is the deep reason CFGs sit strictly below TMs.
  • Q2. Why is the trick to write Ci+1 backwards?
    In the standard construction, alternate configurations are written reversed, so checking whether (Ci, Ci+1) follow by δ becomes a palindrome-like local comparison — pairing up positions that are adjacent across the # boundary. A PDA's stack handles exactly this kind of nested matching.
  • Q3. What's EQCFG?
    EQCFG = {⟨G1, G2⟩ : L(G1) = L(G2)}. Undecidable as an immediate corollary: fix G* with L(G*) = Σ*. Then ⟨G⟩ ∈ ALLCFG iff ⟨G, G*⟩ ∈ EQCFG. A decider for EQCFG would decide ALLCFG.
  • Q4. What is the CFG GM,w roughly like?
    A union of four "violation gadget" sub-grammars: one for each failure mode (bad format, bad start, bad δ-step, bad end). Each gadget is a small CFG that derives strings exhibiting exactly that failure. Union them with G → Gformat | Gstart | Gstep | Gend.
  • Q5. Does the same trick work for LBAs?
    For LBAs we don't need the reverse-write trick because LBAs can do bidirectional sweeps over the whole input. That's the asymmetry between Problem 10 (ELBA via verifying) and this problem (ALLCFG via violating). Both rely on computation histories, but the verification mechanics differ.
