P-A-T-H is N-L-complete. That means every N-L problem reduces to P-A-T-H via a log-space reduction, and P-A-T-H itself is in N-L. The reduction is particularly elegant: take any N-L machine M and input w, and build the configuration graph of M on w. Each configuration is a node, with edges representing single steps. M accepts w if and only if there is a path from the start configuration to the accept configuration in this graph. The graph has polynomial size, and we can construct it in log space because each configuration is described by a log-number of bits. So P-A-T-H is at least as hard as everything in N-L. The widget animates the configuration graph construction and shows the reduction running on a small example. N-L-complete problems are the analog of N-P-complete problems, but at the log-space level.
