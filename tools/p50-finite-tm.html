FINITETM is undecidable

Reduce ATM: build M' whose language is finite iff M does NOT accept w.

CLO-2R1·30 ptsL6
1 · Theory

FINITETM = {⟨M⟩ : L(M) is finite}. Direct application of Rice (Problem 14): "finite" is a nontrivial property of L(M), so FINITETM is undecidable.

Explicit reduction · ATM ≤m ¬FINITETM
M' on input x:
  if |x| ≤ 10 then accept           // finite chunk: ≤ 10
  else run M on w
       if M accepts then accept x

If M accepts w: L(M') = Σ* — infinite. If M doesn't accept w: L(M') ⊆ {x : |x| ≤ 10} — finite. So M accepts w ⇔ ⟨M'⟩ ∉ FINITETM.

3 · Pitfalls
  • Don't conflate "finite" with "small." Finite means there's a bound, no matter how huge. Even 10100 elements is finite.
  • The reduction is to ¬FINITETM. ⟨M, w⟩ ∈ ATM ⇔ L(M') is infinite ⇔ ⟨M'⟩ ∉ FINITETM.
  • The "finite chunk" trick keeps L(M') non-empty in both cases. Without it, "M doesn't accept w" would make L(M') empty, but FINITE includes ∅, so we'd still be fine — the chunk just makes the semantic boundary cleaner.
4 · Q&A defence notes
  • Q1. Why use a fixed finite chunk like {|x| ≤ 10}?
    It ensures L(M') is non-empty in both cases, but finite or infinite depending on M(w). The "10" is arbitrary — any explicit finite set works.
  • Q2. Could we use any finite set?
    Yes — {ε}, {0, 1}, anything explicitly enumerable. The point is the set's finiteness is decidable independently of M.
  • Q3. Apply Rice directly.
    "L(M) is finite" depends only on L(M) (semantic — it doesn't care about M's syntax) and is nontrivial (some TMs have finite languages, some don't). Rice's theorem says any semantic, nontrivial property is undecidable. So FINITETM is undecidable, no construction needed beyond Rice itself.
  • Q4. What's the analog for REGULARTM?
    REGULARTM = {⟨M⟩ : L(M) is regular} is also undecidable by Rice's theorem (regularity is semantic and nontrivial). Construction is similar — make L(M') a specific non-regular language iff M accepts w.
