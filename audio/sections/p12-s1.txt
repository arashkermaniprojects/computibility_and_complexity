Let’s dive into three problems and their deciders, all of which are polynomial-time decidable. First up is the question of whether a given DFA accepts a specific input string. We represent this as A DFA, where you take a DFA B and a string w. The process involves simulating B on w. You start at the initial state, read each symbol in w, and follow the transitions accordingly. After processing the entire string, you accept if you end up in one of the accept states. This runs in linear time with respect to the length of w, which makes it efficient.

Next, we have E DFA, which asks if a DFA B has an empty language. To decide this, you essentially check if you can reach any accept state from the starting state in B’s transition graph. Using a breadth-first search or depth-first search on the graph, you track which states you can reach. This also runs in polynomial time, specifically proportional to the size of B’s transition table.

Finally, consider A CFG, where you determine if a context-free grammar G generates a string w. To solve this, you convert G to Chomsky Normal Form and then employ the CYK algorithm. This fills a table that tracks which variables can derive substrings of w. You accept if the start variable is present in the right spot of the table, and this runs in cubic time relative to the length of w and the size of G. 

So, while all these problems have straightforward yes/no answers, the underlying mechanics differ significantly when you look at their Turing machine counterparts, which are undecidable. Understanding these differences is key to grasping the boundaries of decidability in computation.
