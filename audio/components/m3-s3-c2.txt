Look at this diagram of the LBA B sub M, w that the reduction constructs. The crucial point: M and w are NOT inputs to this LBA — they are hardcoded into its transition table. B sub M, w is a different machine for every (M, w) pair. What IS its input is a candidate computation history x — a string of the form C₁ hash C₂ hash dot dot dot hash C ell, where each Cᵢ is supposed to be one configuration of M.

B sub M, w then runs three local checks on x. Check one: is C₁ the correct start configuration q-zero w? It compares the first chunk against its hardwired copy of w. Check two: does each consecutive pair Cᵢ, Cᵢ-plus-one actually follow by one delta-step of M? This is the 2 by 3 window check from slide one — every local window must obey M's transition function. Check three: does the last configuration C-ell contain the accept state? If all three pass, B sub M, w accepts x; otherwise it rejects.

Each check looks at constant-size windows, so B sub M, w never needs more tape than the input x — that's why it's a legitimate L B A.

Now read the bottom line: L of B sub M, w is non-empty if and only if SOME accepting history exists, which happens if and only if M accepts w. So a decider for E sub L B A would give us a decider for A sub T M — and we know A sub T M is undecidable. Therefore E sub L B A is undecidable too. This is exactly Sipser Theorem 5.10.
