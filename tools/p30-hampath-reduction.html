1 · The gadget reduction · diamonds + spreaders

3SAT ≤p HAMPATH (Sipser §7.5). The reduction builds a graph whose Hamiltonian paths correspond to satisfying assignments. Two pieces:

Variable gadget · the diamond

For each variable xi, build a "diamond" — a long row of nodes connected so that a Hamiltonian path must traverse the diamond either left-to-right (encoding xi = true) or right-to-left (encoding xi = false). No mixed traversal possible.

Clause gadget · the spreader

For each clause C, add a single "clause node" cj. Connect cj to specific diamond positions corresponding to literals appearing in C: if xi appears positively, cj connects to a left-to-right edge in xi's diamond; if ¬xi appears, cj connects to a right-to-left edge.

The Hamiltonian path must visit each cj. The only way to do so without breaking the diamond's required traversal is to use an edge that the assigned literal makes "available" — i.e., an edge in a diamond being traversed in the direction matching the clause's literal. So a Hamiltonian path exists iff every clause has at least one true literal.

3 · Pitfalls
  • The diamond is more than a chain. It has internal structure that forces directional traversal — you can't half-traverse. Sipser's Figure 7.49 shows the precise wiring.
  • Each clause node must be visited exactly once. Hamiltonian = every vertex used once. The spreaders are visited via "side trips" from the diamonds.
  • Gadget reductions look ad hoc but aren't. Every move in the diamond/spreader design serves a precise logical role. Don't be intimidated by the complexity — each piece encodes one logical constraint.

4 · Q&A defence notes
  • Q1. What does "traversal direction = assignment" mean?
    Each variable diamond has two valid Hamiltonian traversals: left-to-right (encoding xi = true) or right-to-left (encoding xi = false). The Hamiltonian path picks one direction per diamond, and the collection of choices defines a Boolean assignment.
  • Q2. How does the clause spreader work?
    For each clause C, the clause node cj connects to specific edges inside the diamonds. Crucially: an edge inside xi's diamond is "available for cj's visit" only when that diamond is being traversed in the direction matching the literal in C. So visiting cj requires that at least one of its literals is true.
  • Q3. Size of the gadget?
    The diamond for each variable has O(m) nodes (to accommodate connections to all clauses), so the whole graph has O(n·m) nodes and O(n·m) edges. Polynomial in formula size.
