Let's dive into some key questions about the single-tape versus two-tape Turing machine models. First, why does the two-tape machine perform so much better on the {0ⁿ1ⁿ} problem? Well, the second tape acts like a scratch register, allowing it to hold half the data. This means the two heads can move past each other in parallel. Instead of making n separate passes, they can count and compare in a single sweep. This "free copying" lets the machine copy all the 0s in linear time and then do the comparison also in linear time.

Next, why doesn't this improvement break the class P? The key is that the simulation cost from a two-tape to a single-tape machine is at most a quadratic increase. Since a quadratic blowup of a polynomial is still a polynomial, P remains intact. 

Now, what's the limit of this multi-tape speedup? Some problems can achieve a linear speedup with extra tapes, while others may only see a polylogarithmic improvement, and some don’t speed up at all. For decision problems, you often get a quadratic-to-linear enhancement, but you’re still bounded by fundamental limits, like Ω(n log n) for sorting.

Finally, this is why the random-access machine model is significant. RAMs allow you to access any cell in constant time without having to shuttle back and forth. They represent what using a laptop feels like, making RAM time polynomially equivalent to Turing machine time, which is why theorists can use either model interchangeably.
