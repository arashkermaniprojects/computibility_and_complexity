1 · Theory
NL-completeness of PATH

For any NL language L with NL machine M, build the configuration graph GM,w: nodes = configurations of M on w, edges = nondet. δ-steps. Then M accepts w ⇔ start config reaches accept config in GM,w. Graph has 2O(log n) = poly(n) nodes.

3 · Q&A defence notes
  • Q1. Why poly(n) configurations?
    A config = state + input pointer + work-tape contents. With O(log n) work-tape, that's O(log n) bits per config × O(1) states. Total 2O(log n) = poly(n) distinct configs.
  • Q2. Why is the reduction in log space?
    We don't materialize the whole graph. We compute neighbours on demand: given a config, list its successors. Each list operation uses O(log n) space.
  • Q3. So NL = {L | L log-space reduces to PATH}?
    Yes. PATH being NL-complete means: NL is precisely the set of problems reducible to PATH via log-space reductions.
