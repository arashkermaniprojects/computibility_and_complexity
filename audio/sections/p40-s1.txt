Let’s dive into why chess is considered hard, especially when we generalize it to larger boards. On a standard 8×8 chessboard, the game has a finite game tree. This means there’s a way to determine the outcome of any position in constant time, though that constant is massive. But when we expand to an n×n board, where n can be any size, the complexity skyrockets. 

In fact, generalized chess on these larger boards is classified as EXPTIME-complete. What this means is that the time it takes to find a solution grows exponentially with the size of the board. Researchers like Fraenkel and Lichtenstein showed how we can encode computations from a Turing machine into chess positions, treating chess pieces as markers of state and the moves as transitions.

Now, if we introduce a bounded-move variant, like saying a game ends in a draw after a set number of moves without a capture, the complexity shifts. This setup makes the problem PSPACE-complete. The limit on moves prevents games from dragging on forever. You'll notice a trend here: many natural games on n×n boards, like generalized Go, also fall into this PSPACE-hard category. It’s a fascinating intersection of game theory and computational complexity!
