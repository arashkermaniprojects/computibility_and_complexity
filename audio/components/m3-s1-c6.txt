This widget is the computation history viewer. Pick a preset, click Load, and you'll see the entire trace of the chosen Turing machine as a table of configurations.

Quick notation refresher. The string q-sub-i bar tells you the machine is in state q-i, and the head sits on the cell immediately to the right of the bar. The bar itself is just a visual delimiter — it is NOT a tape symbol. So a configuration like zero q-one bar zero one represents three tape cells holding zero, zero, one, with the head on the second cell.

Now the crucial thing to understand — what is X? X is a tape symbol the machine writes onto the tape as a private marker. It is NOT part of the input alphabet. The input alphabet for these examples is just zero, but the tape alphabet Gamma also contains X and the blank underscore. The machine uses X to remember that some cell has already been processed.

Let's trace the first preset together, "shift zeros right by one" on input zero zero. We start at step zero: q-zero bar zero zero. State q-zero, head on cell zero, reading a zero. The transition delta of q-zero comma zero equals q-one comma X comma R fires. The machine overwrites cell zero with X, switches to q-one, and moves right. Step one: X q-one bar zero. The head now reads the second zero. The transition delta of q-one comma zero equals q-one comma zero comma R fires — keep the zero, stay in q-one, move right. Step two: X zero q-one bar blank. The head has fallen off the input. The transition delta of q-one comma blank equals q-two comma zero comma L fires. The machine writes a fresh zero in the blank cell — that's the "shift" — switches to q-two, and walks left. Step three: X q-two bar zero zero. Now we have three cells with content X, zero, zero, and the machine starts shuttling back to find the X marker.

Now the second preset, "check parity of zeros" on input zero zero zero zero. The machine simply toggles between two states as it sweeps right. Step zero: q-zero bar zero zero zero zero. Transition delta of q-zero comma zero equals q-one comma zero comma R. The cell stays zero, the state flips to q-one. Step one: zero q-one bar zero zero zero. Transition delta of q-one comma zero equals q-zero comma zero comma R flips back. Step two: zero zero q-zero bar zero zero. After four zeros the machine is back in q-zero, hits a blank, and accepts. If the input had odd length, the machine would end in q-one and reject. So the parity preset never writes X — it only changes the state. The state IS the memory.

Two takeaways: X is a working symbol the machine writes for its own bookkeeping, and only one cell ever changes per step — the cell the head was reading.
