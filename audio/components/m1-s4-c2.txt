Here is how to read the symmetric-difference widget step by step. Step one: load two DFAs, A and B. The widget draws both, side by side. Step two: hit Build C. The widget constructs the symmetric-difference DFA — the language that A accepts XOR the language that B accepts. Picture it as two circles in a Venn diagram with the overlap erased; C accepts exactly the crescent regions. Step three: hit Test emptiness. The widget runs the graph-reachability decider from the previous slide on C. If no accept state of C is reachable from the start state, then L of C is empty, which means A and B are equivalent. If some accept state is reachable, that reachable accept state corresponds to a witness string on which A and B disagree.
