This figure unpacks the recursion theorem statement in three steps.

Step one. You hand me any two-input template T. T is just a program you wrote. Its first slot expects the source code of some Turing machine. Its second slot expects an input string. Inside T, you can do anything you want with those two things. For example, T could be: "print whatever is in slot one and halt." Or it could be: "simulate the machine in slot one on the input in slot two, then flip the accept-reject answer." T is your design choice.

Step two — the puzzle. We want a one-input machine R whose behavior on input w is exactly the same as running T with R's own source code, ⟨R⟩, placed in slot one. Read that again: R's behavior should depend on R's own description. But ⟨R⟩ is the source code of the very machine we're trying to build. R does not have its own source code lying around. Where could it come from? This sounds impossible — a chicken-and-egg paradox.

Step three — the answer. The recursion theorem gives you R, built in two phases. Phase a is a self-print phase. R uses the A-times-B construction from the previous diagram to mechanically assemble its own description ⟨R⟩ onto the tape. It does not read ⟨R⟩ from anywhere — it CONSTRUCTS it from scratch. Phase b then takes that freshly-built ⟨R⟩, plus the original input w, and feeds them into T as slot one and slot two. R outputs whatever T outputs. The net behavior is exactly R-of-w equals T applied to angle-bracket-R-angle-bracket and w. R "knows" its own source code not because it read it, but because the A-B construction computed it.

Why is this called recursion? In any programming language, recursion needs a way for a function to reference itself. The recursion theorem proves that Turing machines have this same superpower. Two consequences: pick T to be "print slot one," and R becomes a quine, a program that prints its own source. Pick T to be "simulate slot one, flip the answer," and R becomes a self-contradicting machine — exactly the tool needed for slick undecidability proofs.
