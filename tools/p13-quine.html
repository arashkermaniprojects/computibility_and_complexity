Quine · a program that prints itself

The cleanest demonstration of the recursion theorem.

CLO-1R1·30 ptsL8
1 · Theory

A quine is a program that, when executed, prints its own source code. Quines exist in every Turing-complete language; their existence is the simplest consequence of the recursion theorem.

The data/code split

A quine has two parts: a string S containing the source with a placeholder, and a printer that emits S with the placeholder replaced by a quoted copy of S itself. Because the printer's behaviour is described in S, the output reproduces the source.

This is the construction underlying the recursion theorem: any TM can effectively obtain its own description.

3 · Q&A defence notes


    

  • Q1. Isn't this cheating — reading its own source file?
    No file I/O. The program only operates on the string variable S, which happens to encode the source. The match is forced by the construction.
    • 

  • Q2. How does this relate to the recursion theorem?
    The recursion theorem says any TM can effectively access its own description. The quine is the special case where the TM uses that description to print itself.
    • 

  • Q3. Why is the recursion theorem useful for proving undecidability?
    It lets a hypothetical TM M reason about its own description ⟨M⟩. We construct M to do the opposite of what a hypothetical decider D(⟨M⟩) says — the standard diagonal argument made cleaner.
    • 

  • Q4. Write the simplest quine you know.
    Lisp: ((lambda (x) (list x (list (quote quote) x))) (quote (lambda (x) (list x (list (quote quote) x))))). Python: s='s=%r;print(s%%s)';print(s%s).
