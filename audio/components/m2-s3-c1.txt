Why is the diagonal cell the engine of the contradiction? Here is the punch line. The contradiction machine D, when run on its own description, asks H whether D accepts angle-D. If H says yes, D rejects. If H says no, D accepts. Either branch makes D contradict H exactly on the diagonal input — the input where D is being fed itself. That is the only place we need the contradiction, but it is enough, because H was supposed to work on every input. One bad cell breaks the whole table. This is the same move as Cantor's flip-the-diagonal, transported from functions on the naturals to Turing machines on encodings.
