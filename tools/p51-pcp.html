The Post Correspondence Problem (PCP) is undecidable

Match dominoes so top = bottom. Surprisingly equivalent to TM acceptance.

CLO-2R1·30 ptsL7
1 · Theory

PCP: given a finite set of dominoes (top/bottom string pairs), is there a finite sequence (with repetition allowed) such that the concatenation of tops equals the concatenation of bottoms?

Theorem · PCP is undecidable

Reduce ATM to PCP using computation histories. Each domino encodes one step of the TM: top accumulates configuration Ci, bottom accumulates Ci+1. A match exists iff the TM has an accepting computation history.

3 · Q&A defence notes
  • Q1. Why is PCP undecidable but small instances solvable by hand?
    Searching for a match is unbounded — there's no polynomial-size witness in general. For small instances BFS terminates; in general it doesn't.
  • Q2. How does the reduction work?
    For each TM transition δ, create a domino: top has Ci portion, bottom has Ci+1 portion. Matches reproduce valid computation histories. Add starter/finisher dominoes to align.
  • Q3. What about the modified PCP that fixes the first domino?
    MPCP is also undecidable; it's the natural target of the TM-history reduction. Standard PCP reduces from MPCP via padding.
