To understand why simulating a nondeterministic Turing machine is non-trivial, let’s break down what happens when you try to simulate it directly. Imagine you have a nondeterministic machine N that uses f(n) space. Now, this machine can have at most 2 raised to the O of f(n) distinct configurations. These configurations come from the combination of its current state, the contents of the tape, and the position of the tape head. If N is going to halt, it needs to do so within a number of steps related to this exponential count of configurations.

Now, if you tried to simulate N by simply exploring all these configurations using a breadth-first search, you’d quickly run into a problem. The number of reachable configurations is exponential, which means you would need linear space in that number. This breaks our space bounds. So, what do we do instead? We can reformulate our approach. We create a reachability function that asks whether one configuration can reach another within a certain number of steps. This way, we can turn the problem into a recursive one that’s much easier to handle. The key is to use this recursive structure cleverly to keep our space requirements manageable.
