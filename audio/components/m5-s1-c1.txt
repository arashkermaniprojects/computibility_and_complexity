This is the Cook-Levin tableau. Read it top to bottom: time flows downward, tape position flows across. Each row is a complete snapshot of the verifier's tape at one moment. The cell at row t, column p, records which symbol sits at tape position p at time t. The very first row shows the start configuration: the head sits at cell zero in state q zero, the input fills cells one through three, and the guessed certificate occupies cells five through seven. That dashed orange rectangle marks any two by three window — two consecutive rows times three consecutive columns. Because the Turing machine's head moves at most one cell per step, what happens at row t plus one, column p, is fully determined by the three cells directly above it. So if every two-by-three window obeys the transition function delta, the entire computation is legal. Each window becomes a small SAT clause. The green q-accept box near the bottom encodes the requirement that the accept state appear somewhere in the tableau — one big OR clause across every position. Putting it all together: the boolean variables count one for each combination of time, position, and symbol, and the total formula is polynomial in the input size. The formula is satisfiable if and only if some certificate makes the verifier accept. That single construction proves SAT is N P hard.
