Self-aware Turing machines

A TM that reads its own description and acts on it. Used to give a one-line proof that ATM is undecidable.

CLO-1R1·30 ptsL8
1 · Theory

The recursion theorem says: any TM can effectively obtain its own description ⟨M⟩ on its tape and use it as data. This gives a beautifully short re-proof of ATM-undecidability.

ATM undecidable · recursion-theorem proof

Suppose D decides ATM. By the recursion theorem, build a TM M that on input w:

  1. Obtain ⟨M⟩.
  2. Compute D(⟨M, w⟩).
  3. Output the OPPOSITE of D's answer.

Then M accepts w ⇔ D says M doesn't accept w ⇔ M doesn't accept w. Contradiction.

The diagonalization is now hidden inside M itself, not stretched across a separate machine D' as in Problem 3.

3 · Q&A defence notes
  • Q1. Compare with the Problem 3 proof.
    Problem 3 builds an external D that flips H. Problem 16's M flips D from inside itself, using the recursion theorem to get ⟨M⟩. Same contradiction, cleaner structure.
  • Q2. Does M actually "know" its own code?
    In the sense the recursion theorem provides: yes, ⟨M⟩ is written on M's tape before computation begins (or constructable on demand). M can then operate on ⟨M⟩ like any other input.
  • Q3. Where else is self-reference used in this course?
    Rice's theorem (P14), Gödel's incompleteness, the construction of complete problems in complexity theory, fixed-point combinators in λ-calculus.
