Let’s dive into some Q&A about 2-SAT and its properties. First off, why does 2-SAT have this efficient algorithm while 3-SAT doesn’t? Well, think about how a 2-clause like “a or b” breaks down into two clear implications: if ¬a is true, then b must be true, and vice versa. The structure of these implications creates a nice tree-like graph that’s manageable. But with a 3-clause like “a or b or c,” you end up with six implications that don’t mesh together neatly. This complexity makes it tough to apply the same graph trick effectively.

Now, why do we say 2-SAT is in NL and not L? When we’re checking for a path in the implication graph, we use nondeterminism, which means we can explore multiple possibilities at once. If we could check that path deterministically in L, it would imply that L equals NL, but that’s still an open question in computer science.

Is 2-SAT in P? Absolutely! We can compute strongly connected components in linear time using algorithms like Tarjan’s or Kosaraju’s. This gives us both the decision and the assignment efficiently. Lastly, Horn-SAT, which has at most one positive literal per clause, is also in P thanks to unit propagation, making it another "easy" case of SAT. So, both 2-SAT and Horn-SAT sit comfortably in the realm below NP-complete problems.
