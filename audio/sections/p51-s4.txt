Let’s dive into some important questions about the Post Correspondence Problem, or PCP. First, why is PCP undecidable, yet we can solve small instances by hand? The key is that while searching for a match can be done with brute force for small cases, the search space can grow enormously with larger instances. There’s no polynomial-time witness to guarantee a solution, so while you might find matches quickly in smaller cases, larger problems can lead to infinite depths in the search.

Next, let’s talk about how the reduction from Turing machines to PCP works. For every transition in a Turing machine, you create a domino where the top part represents the current state and the bottom part represents the next state. When you match these dominos, you’re effectively building a computation history. To start things off, you need a starter domino for the initial state and another for the accept state to make sure everything aligns correctly.

Now, onto modified PCP, or MPCP. This version fixes the first domino in the sequence, and interestingly, it's also undecidable. The reduction from a Turing machine to MPCP is straightforward since the first domino is predetermined. You can even reduce standard PCP from MPCP using a padding technique that simulates the constraint of the starter domino.

Lastly, is PCP a useful tool for proving undecidability in other problems? Absolutely! Many natural problems can be reduced from PCP. The concept of matching two strings through combinatorial choices appears in various contexts, such as context-free language ambiguity, post-canonical systems, and different tiling problems. So, understanding PCP opens the door to a wide range of undecidability proofs!
