Here is the bookkeeping. The single-tape Turing machine for the language zero to the n one to the n has to match each zero with a one, and to match a pair it sweeps across the tape. One sweep costs order n steps. There are n over two pairs to match. So the total time is order n squared. The step counter in the widget grows quadratically as you push the input length up. This is the canonical example of a problem that lives in P but is not linear, and it is the example Sipser uses to motivate the two-tape speed-up on the next slide.
