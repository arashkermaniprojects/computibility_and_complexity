Two examples to make the recursion theorem concrete.

Example one: the quine. Pick the template T to be the program "print whatever's in slot one and halt." T just echoes its first argument. Plug T into the recursion theorem. You get back a one-input machine R with the behavior: R-of-w prints R's own description and halts. That's a quine — a Turing machine that prints its own source code. The recursion theorem says quines exist in any programming model that's at least as powerful as a Turing machine. The widget below this section walks through the construction of an actual JavaScript quine.

Example two: a slick proof that A-T-M is undecidable. Suppose, for contradiction, some Turing machine D decides A-T-M — given any pair of a machine and a string, D answers whether the machine accepts the string. Pick the template T to be the program "run D on slot one and slot two, then flip the answer." Now plug T into the recursion theorem. You get back R with the behavior: R-of-w runs D on the pair R and w, then flips. But that means R accepts w if and only if D says R does NOT accept w — a direct logical contradiction. So D cannot exist. The recursion theorem lets us skip the awkward self-reference dance of the original diagonalization proof: instead of constructing a separate contradiction machine and arguing it must equal itself, we let R BE the contradiction machine directly, because the recursion theorem guarantees such an R exists.
