When we talk about reductions in the context of nondeterministic logarithmic space, or NL, we have to be careful. The reductions we use need to work within log space, not polynomial time. That’s crucial because if we apply a polynomial-time reduction, we might end up generating an excessive number of configurations. Remember, configurations in NL are pretty short, requiring only O(log n) bits. If we were to create exponentially many configurations, that would be a problem, as 2 to the power of O(log n) is still polynomial in size.

So, when we say to avoid materializing the whole graph, it’s because doing so would require polynomial space, which goes against the log-space constraint we’re working with. Instead, we compute adjacencies on demand, which keeps everything efficient and within the limits of what log space can handle. This way, we respect the principles of NL-completeness, ensuring that our reductions and computations stay true to the space limitations we’re dealing with.
