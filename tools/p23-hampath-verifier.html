HAMPATH ∈ NP · verifying a Hamiltonian path

Certificate = a vertex sequence; verifier checks length, distinctness, edges.

CLO-3R2·30 ptsL10
1 · Theory

HAMPATH = {⟨G, s, t⟩ : G has a Hamiltonian path from s to t}. NP-complete. Verifier is easy; finding the path seems hard.

Verifier in poly time

Given (G, s, t, π = v₁ v₂ … v_n): check |π| = n, all distinct, v₁ = s, v_n = t, every (v_i, v_{i+1}) is an edge. O(n²).

3 · Q&A defence notes
  • Q1. Why O(n²)?
    For n vertices: n length checks, n² distinctness checks (or n with a hash), n−1 edge lookups.
  • Q2. Why is finding hard?
    Brute force tries n! paths. No poly-time algorithm known. HAMPATH is NP-complete (3SAT reduces to it).
  • Q3. What's the certificate size?
    n vertex labels, each O(log n) bits. Total O(n log n) — polynomial in input size.
