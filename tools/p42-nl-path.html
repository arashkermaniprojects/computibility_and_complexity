PATH ∈ NL · nondeterministic guess-the-path

Guess a neighbour each step; log n bits suffice per branch.

CLO-5R3·40 ptsL15
1 · Theory
NL algorithm for PATH
v ← s; counter ← 0
while counter ≤ |V|:
   if v = t accept
   v ← nondet. choice from neighbours of v
   counter += 1
reject

Memory: v (log n bits) + counter (log n bits) = O(log n). Hence PATH ∈ NL.

3 · Q&A defence notes
  • Q1. Why nondeterministic and not deterministic?
    A deterministic log-space machine doesn't have enough memory to remember which neighbours have been tried. Nondeterminism lets us "magically" guess the right neighbour.
  • Q2. Why must counter ≤ |V|?
    If a path exists, it has length ≤ |V|−1 (no repeats needed). So we cap to avoid infinite loops.
  • Q3. Is PATH in L?
    Open! Showing PATH ∈ L is equivalent to L = NL, one of the major open questions.
