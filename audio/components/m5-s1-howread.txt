How to read the Cook-Levin tableau. The grid is the verifier's computation tableau. Time flows downward, tape position runs across. Cell at row t, column p, records what symbol sits on tape position p at time step t — and whether the head and state are there. Each row is a complete configuration of the verifier V at one instant. Row 0 is the start configuration: head at cell 0 in state q-zero, the input fills cells 1 through 3, and the guessed certificate fills cells 5 through 7. The dashed orange 2-by-3 window captures one tiny step of the TM — because the head moves at most one cell per step, what cell at time t+1, position p becomes depends only on the three cells directly above. So if every 2-by-3 window obeys the transition function delta, the whole evolution is a legal run. Each window becomes a small SAT clause. The green q-accept box near t equals k encodes the requirement that the accept state appear somewhere — a single big OR-clause. The boolean variables x-sub-t-p-s are true if cell (t,p) holds symbol s. Three clause families — start, move, accept — combine into phi-x. Satisfiable if and only if some certificate makes V accept. That means every N P problem reduces to sat in polynomial time.
